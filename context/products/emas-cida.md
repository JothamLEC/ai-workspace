# EMAS CIDA

## What is EMAS CIDA?

**EMAS Customer ID Assurance (CIDA)** is a flexible, customizable, and future-proof framework that allows digital businesses to scale their Assurance, Authentication, and Authorization (3A) requirements.

## What EMAS CIDA Aims to Achieve

Many digital businesses today have rapidly expanding 3A requirements:

- **Evolving security needs** - Usernames and passwords are no longer sufficient. Users now need security questions, OTP codes, and more. eKYC systems alone may not meet future demands.
- **Constant innovation by fraudsters** - The 3A space is evolving quickly as fraudsters find new attack vectors.
- **Demand for automation** - Businesses require more automation and digitalization within their processes.
  - Example: Banks need systems that automatically retrieve user credit scores during onboarding, reducing processing time.

In addition to EMAS eKYC, Innov8tif partners with regional providers to help businesses implement this comprehensive framework.

## The Four Layers of EMAS CIDA

### 1. Portal Access Layer
Enables businesses to administer and operate EMAS CIDA. Operations personnel can review transactions performed, API usage, and more.

**Learn more:** [Portal Access Layer documentation](../features/portal-access-layer.md)

### 2. Channel Layer
Determines how functions and features are delivered and connected to each other. Uses APIs and robotic process automation to perform 3A tasks.

### 3. Features Layer
Highlights the various technologies needed to perform 3A tasks:

#### ID Proofing
Collecting user data to ensure they are genuine:
- [EMAS eKYC](./emas-ekyc.md) - Core identity verification
- [Digital Footprint Analysis](../features/digital-footprint-analysis.md) - Synthetic identity fraud detection
- [Video Call Verification](../features/video-call-verification.md) - Human-assisted verification

#### Customer Due Diligence
Determining if users are qualified for business services:
- [Financial Risk Checks](../features/financial-risk-checks.md) - Credit and financial screening
- [Income & Address Proofing](../features/income-address-proofing.md) - Document verification
- [Biometric Alert List](../features/biometric-alert-list.md) - Watchlist screening

#### ID Authentication
Ensuring the user is authorized to perform transactions:
- [Device Binding](../features/device-binding.md) - Secure device registration
- [Biometric Authentication](../features/biometric-authentication.md) - Biometric login
- [Device Risk Screening](../features/device-risk-screening.md) - Device security checks

### 4. Data Layer
Forms the foundation of the EMAS CIDA framework, where customer information is securely stored for use throughout the customer lifecycle.

## Industry Use Cases

### Banking
24/7 digital banking services. Secure customer onboarding and transaction validation including:
- Interbank transfers
- Loan applications
- Credit card applications

### Telecommunications
Due diligence for high-value services:
- Phone plan subscriptions
- SIM card registration
- Account transfers

### E-commerce
Identity verification and fraud prevention:
- Customer verification during checkout
- Preventing unauthorized purchases
- Sign-up abuse prevention

### Healthcare
Secure authentication for medical services:
- Telemedicine consultations
- Online appointment booking
- Patient privacy protection
- Medical identity theft prevention

### Insurance
Policy and claims management:
- Customer identity verification during applications
- Claims processing
- Fraud prevention
- Ensuring only eligible customers access services
